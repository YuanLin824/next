# SEO 介绍

Search Engine Optimization，即搜索引擎优化。

是一种通过优化网站结构和内容，提高网站在搜索引擎中的排名，从而吸引更多流量和用户的策略。

SEO 是一个长期优化过程（一般优化 1-3 个月才能看到效果），无需急于求成。

## 黑帽 SEO

黑帽 SEO 是指通过不正当手段，如关键词堆砌、隐藏文本、欺诈性链接等，来提高网站在搜索引擎中的排名。

## 白帽 SEO

白帽 SEO 是通过正当技术手段来提高网站在搜索引擎中的排名：

- 优化 TDK
- 优化网站结构
- 优化 robots.txt
- 优化 sitemap.xml
- 优化 JSON-LD
- 优化 Open Graph
- 优化 Web Vitals 等

### robots.txt

`robots.txt` 告诉搜索引擎爬虫哪些页面可以抓取、哪些不可以。

Next.js App Router 中，在 `app` 目录下创建 `robots.ts` 文件：

```ts
// app/robots.ts
import type { MetadataRoute } from "next"

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: "*",
      allow: "/",
      disallow: ["/api/", "/admin/"],
    },
    sitemap: "https://example.com/sitemap.xml",
  }
}
```

### sitemap.xml

站点地图帮助搜索引擎发现和索引你的所有页面。

```ts
// app/sitemap.ts
import type { MetadataRoute } from "next"

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: "https://example.com",
      lastModified: new Date(),
      changeFrequency: "weekly",
      priority: 1,
    },
    {
      url: "https://example.com/about",
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 0.8,
    },
  ]
}
```

静态路由直接返回数组；动态路由可调用 API 生成：

```ts
// app/blog/[slug]/sitemap.ts
export async function generateSitemaps() {
  // 获取所有博客文章
  const posts = await fetch("https://api.example.com/posts").then((res) => res.json())
  return posts.map((post) => ({
    url: `https://example.com/blog/${post.slug}`,
    lastModified: new Date(post.updatedAt),
  }))
}
```

### TDK + Meta

T（Title）、D（Description）、K（Keywords）是页面 SEO 的核心三要素。Next.js 通过 [Metadata API](https://nextjs.org/docs/app/building-your-application/optimizing/metadata) 优雅地管理。

```tsx
// app/layout.tsx — 全局默认
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: {
    default: "前端 Blog",
    template: "%s | 前端 Blog",
  },
  description: "Next.js 全栈应用模板，React 19 + TypeScript + shadcn/ui",
  keywords: ["Next.js", "React", "TypeScript", "全栈"],
  authors: [{ name: "Yuan Lin" }],
}
```

```tsx
// app/blog/[slug]/page.tsx — 动态页面
import type { Metadata } from "next"

export async function generateMetadata({
  params,
}: {
  params: { slug: string }
}): Promise<Metadata> {
  const post = await getPost(params.slug)
  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.cover],
    },
  }
}
```

**HTML 语义化标签** 也是 SEO 的重要一环：

- `<header>` / `<nav>` / `<main>` / `<article>` / `<section>` / `<aside>` / `<footer>`
- 合理的标题层级（h1 → h2 → h3，不跳级）
- 图片添加有意义的 `alt` 属性

### JSON-LD

[JSON-LD](https://json-ld.org/) 是 Google 推荐的**结构化数据**格式，能让搜索引擎更好理解页面内容，并在搜索结果中展示富文本摘要（Rich Snippets）。

```tsx
// app/blog/[slug]/page.tsx
export default async function Post({ params }: { params: { slug: string } }) {
  const post = await getPost(params.slug)

  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "BlogPosting",
    headline: post.title,
    description: post.excerpt,
    datePublished: post.createdAt,
    dateModified: post.updatedAt,
    author: {
      "@type": "Person",
      name: "Yuan Lin",
    },
  }

  return (
    <>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      <article>
        <h1>{post.title}</h1>
        {post.content}
      </article>
    </>
  )
}
```

常用的 Schema 类型：`BlogPosting`、`Article`、`Product`、`Organization`、`BreadcrumbList`、`FAQPage`。

### Open Graph (OG)

[Open Graph](https://ogp.me/) 控制网站在社交媒体（Twitter/X、Facebook、LinkedIn 等）分享时展示的卡片样式。

```tsx
// app/layout.tsx
export const metadata: Metadata = {
  openGraph: {
    title: "前端 Blog",
    description: "Next.js 全栈应用模板",

    images: [
      {
        url: "https://example.com/og.png",
        width: 1200,
        height: 630,
      },
    ],
    locale: "zh_CN",
    type: "website",
  },
  twitter: {
    card: "summary_large_image",
    title: "前端 Blog",
    description: "Next.js 全栈应用模板",
    images: ["https://example.com/og.png"],
  },
}
```

动态 OG 图片可使用 `ImageResponse`：

```tsx

```

### Web Vitals

[Web Vitals](https://web.dev/vitals/) 是 Google 提出的网页性能核心指标，直接影响搜索排名。

| 指标 | 全称                      | 目标值  | 说明             |
| ---- | ------------------------- | ------- | ---------------- |
| LCP  | Largest Contentful Paint  | ≤ 2.5s  | 最大内容绘制时间 |
| FID  | First Input Delay         | ≤ 100ms | 首次输入延迟     |
| INP  | Interaction to Next Paint | ≤ 200ms | 下一次绘制的交互 |
| CLS  | Cumulative Layout Shift   | ≤ 0.1   | 累计布局偏移     |
| TTFB | Time to First Byte        | ≤ 0.8s  | 首字节时间       |

Next.js 内置优化帮你自动达成大部分指标：

- **LCP**: `next/image` 自动优化图片，`loading="eager"` 预加载首屏大图
- **CLS**: `next/image` 自动预留宽高空间，`next/font` 防止字体偏移
- **TTFB**: SSR 服务端直出 HTML，CDN 边缘缓存

使用 `useReportWebVitals` 埋点上报：

```tsx
// app/web-vitals.ts
"use client"

import { useReportWebVitals } from "next/web-vitals"

export function WebVitals() {
  useReportWebVitals((metric) => {
    console.log(metric)
    // 发送到分析服务：如 Google Analytics、Vercel Analytics
  })
}
```

# 国际化

国际化 (Internationalization) 是 Next.js 提供的一种机制, 用于支持多语言的网站

例如可以实现中英文切换, 包括接口同步翻译, 以及不同语言所展示的页面不一样等

i18n (internationalization) 的由来是取自开头 i 和结尾 n 的中间有 18 个字母, 所以称为 i18n

k8s (kubernetes) 就是取自开头 k 和结尾 s 的中间有 8 个字母, 所以称为 k8s

## 国际化术语解释

language: 语言(排名不分先后) [语言速查表](https://zh.wikipedia.org/wiki/ISO_639-1)

1. 英语 (English) 用 en 表示
2. 中文 (Chinese) 用 zh 表示 (zh 取自 Zhongwen 的拼音)
3. 日本语 (Japanese) 用 ja 表示
4. 韩国语 (Korean) 用 ko 表示

territory：地区(排名不分先后) [地区速查表](https://zh.wikipedia.org/wiki/ISO_3166-1)

1. 美国 (United States) 用 US 表示
2. 中国 (China) 用 CN 表示
3. 日本 (Japan) 用 JP 表示
4. 韩国 (Korea) 用 KR 表示

一般把语言和地区组合起来, 称为 locale, 例如 en-US 表示英语 (美国), zh-CN 表示中文 (中国)

解析规则:

| 语言  | 权重 (q值) | 说明          |
| ----- | ---------- | ------------- |
| zh-CN | 1.0        | (默认) 最优先 |
| zh    | 0.9        | 第二优先      |
| en-US | 0.8        | 第三优先      |
| en    | 0.7        | 第四优先      |
| ja    | 0.6        | 最低优先      |

## 国际化实现原理

- Next.js 建议使用 http 报文头来判断用户使用的语言 Accept-Language
  - Accept-Language: zh-CN,zh;q=0.9,en;q=0.8
    - 表示用户使用中文 (中国), 如果用户没有设置, 则使用默认语言

- Accept-Language 规则
  - zh-CN,zh;q=0.9,en;q=0.8
    - 表示用户使用中文 (中国), 权重为 1 (最大值 1 会被省略), 中文 (中国) 权重为 0.9, 英语 (美国) 权重为 0.8

```ts
// npm i negotiator # 用于解析 Accept-Language
// npm i @formatjs/intl-localematcher # 用于匹配语言

// Accept-Language: zh-CN,zh;q=0.9,en-US;q=0.8,en;q=0.7,ja;q=0.6

// i18n/index.ts
export const locales = ["en", "zh", "ja", "ko"] // 配置项目支持的语言
export const defaultLocale = "zh" // 项目默认使用的语言

// src/proxy.ts
import { NextRequest, NextResponse } from "next/server"
import Negotiator from "negotiator"
import { match } from "@formatjs/intl-localematcher"
import { locales, defaultLocale } from "@/i18n/index" // 导入项目支持的语言和默认语言
export default function proxy(req: NextRequest, res: NextResponse) {
  // 如果请求路径为根路径, 则直接返回 首页不做任何处理
  if (req.nextUrl.pathname === "/") {
    return NextResponse.next()
  }
  // 如果路径已经包含所支持的语言, 则直接返回 例如 /zh/about /zs/home 等
  if (locales.some((locale) => req.nextUrl.pathname.startsWith(`/${locale}`))) {
    return NextResponse.next()
  }
  // 获取请求头
  const headers = {
    "accept-language": req.headers.get("accept-language") || "",
  }
  // 解析请求头
  const negotiator = new Negotiator({ headers })
  // 获取语言
  const language = negotiator.languages()
  // ['zh-CN', 'zh', 'en-US', 'en', 'ja']  // 按优先级从高到低排序
  const lang = match(language, locales, defaultLocale)
  // language-浏览器支持的语言 locales-项目支持的语言 defaultLocale-项目默认语言
  // 匹配语言例如 zh-CN 则 lang 返回 zh en-US 则 lang 返回 en 如果没有匹配到则返回默认语言 defaultLocale
  const pathname = req.nextUrl.pathname
  // 拼接语言
  req.nextUrl.pathname = `/${lang}${pathname}`
  // 重定向 例如用户访问的是/home 我们则读取语言后重定向到 /zh/home 给它增加语言前缀
  return NextResponse.redirect(req.nextUrl)
}
export const config = {
  matcher: [
    "/((?!api|_next/static|_next/image|favicon.ico).*)", // 跳过内部匹配路径
  ],
}
```

# NextJS 配置

`next.config.ts` 是 Next.js 项目的核心配置文件，位于项目根目录。

## 基础配置 (NextJS)

```ts
// next.config.ts
import type { NextConfig } from "next"

const nextConfig: NextConfig = {
  reactCompiler: true, // 启用 React Compiler 自动优化
  reactStrictMode: true, // React 严格模式
  pageExtensions: ["js", "jsx", "ts", "tsx"], // 页面扩展名
  distDir: "dist", // 构建输出目录（默认 .next）
  trailingSlash: true, // URL 尾部斜杠
}

export default nextConfig
```
