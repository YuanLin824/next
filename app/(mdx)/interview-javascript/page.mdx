# JavaScript

## JavaScript 数据类型

- JS 数据类型主要分为两大类
- 两者主要区别是存储位置不同
- 简单数据类型存储在栈中 (复制时, 是创建一份新的数据)
- 复杂数据类型存储在堆中 (堆中的数据只有一份, 复制时, 复制的是指向数据的地址)

### 简单数据类型

- Number, String, Boolean, Undefined, Null, Symbol

### 复杂/引用数据类型

- Object, Array, Function

```js
// 区别
// typeof 和 instanceof 都是判断数据类型的方法
// typeof 返回一个字符串, instanceof 返回一个布尔值
// instanceof 可以准确的判断引用数据类型, 不能正确判断基础数据类型
// typeof 不能判断引用数据类型

// instanceof 实现原理
function myInstanceof(left, right) {
  // 基础数据类型直接返回 false
  if (typeof left !== "object" || left === null) return false
  // getPrototypeOf 是 Object 对象自带的 API, 能够拿到参数的原型对象
  let proto = Object.getPrototypeOf(left)
  while (true) {
    if (proto === null) return false
    // 找到相同原型对象, 返回 true
    if (proto === right.prototype) return true
    proto = Object.getPrototypeOf(proto)
  }
}

// 通用的数据类型判断方法
function getType(obj) {
  let type = typeof obj
  if (type !== "object") {
    return type
  }
  return Object.prototype.toString.call(obj).replace(/^\[object (\S+)\]/, "$1")
}
```

## 原型 & 原型链

- 对象有 \_\_proto\_\_ 属性, 指向它构造函数的原型对象
- 函数有 prototype 属性, 指向函数的原型对象

- 构造函数: Person
- 实例: person
- 原型: Person.prototype

- person = new Person()
- Person === Person.prototype.constructor
- person.\_\_proto\_\_ === Person.prototype

- person.\_\_proto\_\_ === Person.prototype
- Person.prototype.\_\_proto\_\_ === Object.prototype
- Object.prototype.\_\_proto\_\_ === null
- 这样形成的链式结构是原型链

## var & let & const

### 声明

- var: 可以重复声明, 变量的值会被更新
- let: 不能重复声明
- const: 不能重复声明, 且生命时必须初始化

### 变量提升

- var: 声明的变量会被提升到作用域顶部, 但初始化不会提升 (即可以在声明之前访问, 但值是 undefined)
- let, const: 声明的变量也会被提升, 在初始化之前处于 "暂时性死区"(Temporal Dead Zone, TDZ); 即在声明之前访问会报错

### 作用域

- var: 全局作用域, 函数作用域
- let, const: 全局作用域, 函数作用域, 块作用域

## 作用域 & 作用域链

### 作用域: 变量 (变量作用域又称上下文) 和函数生效 (能被访问) 的区域集合

### 作用域分类

- 全局作用域
- 函数作用域
- 块作用域

访问一个变量时, 从当前作用域向外部作用域查找, 直到全局作用域, 这样形成的一个链式结构称为作用域链

## JavaScript 中的执行上下文和执行栈

### 执行上下文 (即作用域)

- 全局执行上下文: 只有一个, 浏览器中的全局对象是 window 对象, this 指向这个全局对象
- 函数执行上下文: 存在无数个, 只有函数被调用时才会创建, 每次调用都会创建一个新的执行上下文
- Eval 函数执行上下文: 指 eval 函数中的代码 (不建议使用)

### 执行栈 (调用栈)

- LIFO (后进先出) 结构, 用于存储代码执行期间创建的所有执行上下文
- JS 引擎执行第一行代码时, 会创建一个全局执行上下文并压入执行栈
- JS 引擎碰到一个函数时, 会创建一个函数执行上下文并压入执行栈; 函数执行结束后, 对应的上下文会被弹出

## new 操作符

```js
// new 操作符做了那些工作
export function myNew(Func, ...args) {
  // 1. 创建一个空对象
  const obj = {}
  // 2. 将新对象的 __proto__ 属性指向构造函数的原型
  Object.setPrototypeOf(obj, Func.prototype)
  // 3. 将构造函数的 this 指向新对象
  const result = Func.apply(obj, args)
  // 4. 根据返回值判断
  return result instanceof Object ? result : obj
}

// ------------------------------ 测试 ------------------------------
function Person(name, age) {

    name,
    age,
    greet() {
      return `Hello, I'm ${this.name}`
    },
  }
}

const p = myNew(Person, "Alice", 25)
console.log(p.name) // "Alice"
console.log(p.age) // 25
console.log(p.greet()) // "Hello, I'm Alice"
console.log(p instanceof Person) // true
```

## apply & call & bind

- apply(this, args[]) 改变 this 后, 立即执行, 临时改变
- call(this, ...args) 改变 this 后, 立即执行, 临时改变
- bind(this, ...args) 返回改变 this 后的函数, args 可以分多次传入

| 特性     | call               | apply                | bind                 |
| -------- | ------------------ | -------------------- | -------------------- |
| 执行时机 | 立即执行           | 立即执行             | 返回新函数, 延迟执行 |
| 参数形式 | 展开参数 `1, 2, 3` | 数组参数 `[1, 2, 3]` | 展开参数, 可柯里化   |
| 支持 new | ❌                 | ❌                   | ✅                   |

```js
// ------------------------------ apply ------------------------------
Function.prototype.myApply = function (thisArg, argsArray) {
  const fn = this
  thisArg = thisArg || globalThis
  const symbolKey = Symbol("fn")
  thisArg[symbolKey] = fn
  // apply 的参数是数组形式
  const result = argsArray ? thisArg[symbolKey](...argsArray) : thisArg[symbolKey]()
  delete thisArg[symbolKey]
  return result
}

// ------------------------------ call ------------------------------
Function.prototype.myCall = function (thisArg, ...args) {
  // 1. 获取要执行的函数
  const fn = this
  // 2. 处理 thisArg (null/undefined 指向 window/global)
  thisArg = thisArg || globalThis
  // 3. 将函数作为对象的临时方法
  const symbolKey = Symbol("fn")
  thisArg[symbolKey] = fn
  // 4. 执行并获取结果
  const result = thisArg[symbolKey](...args)
  // 5. 清理临时属性
  delete thisArg[symbolKey]
  return result
}

// ------------------------------ bind ------------------------------
Function.prototype.myBind = function (thisArg, ...bindArgs) {
  const fn = this
  // 返回一个绑定函数
  function boundFn(...callArgs) {
    // 判断是否是 new 调用
    const isNew = this instanceof boundFn
    // new 调用时 this 指向实例, 否则指向绑定的 thisArg
    const context = isNew ? this : thisArg
    // 合并参数: bind 时的参数 + 调用时的参数
    return fn.apply(context, [...bindArgs, ...callArgs])
  }
  // 维护原型链, 使 boundFn 能被 new
  boundFn.prototype = Object.create(fn.prototype)
  return boundFn
}

// ------------------------------ 测试 ------------------------------
const obj = { name: "Alice" }
function greet(greeting, punctuation) {
  return `${greeting}, ${this.name}${punctuation}`
}
// === apply 测试 ===
console.log(greet.myApply(obj, ["Hi", "?"])) // "Hi, Alice?"
// === call 测试 ===
console.log(greet.myCall(obj, "Hello", "!")) // "Hello, Alice!"
// === bind 测试 ===
const boundGreet = greet.myBind(obj, "Hey")
console.log(boundGreet("!!!")) // "Hey, Alice!!!"
// bind + new 测试
function Person(name, age) {
  this.name = name
  this.age = age
}
const BoundPerson = Person.myBind(null, "Bob")
const p = new BoundPerson(25)
console.log(p.name, p.age) // "Bob" 25
console.log(p instanceof Person) // true
```

事件代理/委托:

- 事件委托会把一个或一组元素的事件委托到它的父级或更外层元素上, 真正绑定事件的是外层元素, 而不是目标元素

优点:

- 减少整个页面所需内存, 提升整体性能
- 动态绑定, 减少重复工作

局限性:

- focus, blur 这些事件没有事件冒泡机制, 所以无法委托
- mousemove, mouseout 等事件, 虽然有事件冒泡, 但只能不断通过位置去计算定位, 对性能消耗高, 也不适用于事件委托

## 防抖 (debounce) & 节流 (throttle)

```js
// 防抖: 执行最后一次
function debounce(func, wait) {
  let timeout
  return function (...args) {
    const context = this
    clearTimeout(timeout)
    timeout = setTimeout(() => func.apply(context, args), wait)
  }
}
// 节流: 执行第一次
function throttle(func, wait) {
  let timeout
  return function (...args) {
    if (timeout) return
    clearTimeout(timeout)
    func.apply(this, args)
    timeout = setTimeout(() => (timeout = 0), wait)
  }
}
```

## AJAX 原理 & 实现

AJAX (Async JavaScript and XML/异步 JS 和 XML)

### AJAX 原理

1. 通过 XMLHttpRequest 对象来向服务器发异步请求
2. 获取服务器数据
3. 用 JS 操作 DOM 更新页面

### 实现过程

1. 创建 AJAX 的核心对象 XMLHttpRequest 对象
2. 通过 XMLHttpRequest 对象的 open() 方法与服务端建立连接
3. 构建请求所需的数据内容, 通过 XMLHttpRequest 对象的 send() 方法发送给服务器
4. 通过 XMLHttpRequest 对象的 onreadystatechange 事件监听服务器的通信状态
5. 接受并处理服务端向客户端响应的数据
6. 将结果更新到 HTML 页面中

```js
// 封装 AJAX
function ajax(options) {
  // 创建 XMLHttpRequest 对象
  const xhr = new XMLHttpRequest()
  // 初始化参数内容
  options = options || {}
  options.type = (options.type || "GET").toUpperCase()
  options.dataType = options.dataType || "json"
  const params = options.data
  // 发送请求
  if (options.type === "GET") {
    xhr.open("GET", options.url + "?" + params, true)
    xhr.send(null)
  } else if (options.type === "POST") {
    xhr.open("POST", options.url, true)
    xhr.send(params)
  }
  xhr.onreadystatechange = function () {
    if (xhr.readyState === 4) {
      let status = xhr.status
      if (status >= 200 && status < 300) {
        options.success && options.success(xhr.responseText, xhr.responseXML)
      } else {
        options.fail && options.fail(status)
      }
    }
  }
}
// 使用 ajax
ajax({
  type: "post",
  dataType: "json",
  data: {},
  url: "https://xxxx",
  success(text, xml) {
    // 成功的回调
    console.log(text)
  },
  fail(status) {
    // 失败的回调
    console.log(status)
  },
})
```

```js
// offsetTop, scrollTop
function isInViewport(el) {
  const viewportHeight =
    window.innerHeight || document.documentElement.clientHeight || document.body.clientHeight
  const offsetTop = el.offsetTop
  const scrollTop = document.documentElement.scrollTop || document.body.scrollTop
  const top = offsetTop - scrollTop
  return top <= viewportHeight
}

// getBoundingClientRect
function isInViewport(element) {
  const rect = element.getBoundingClientRect()
  return (
    rect.top >= 0 &&
    rect.left >= 0 &&
    rect.bottom <= (window.innerHeight || document.documentElement.clientHeight) &&
    rect.right <= (window.innerWidth || document.documentElement.clientWidth)
  )
}
```

1. 解析 URL 是否合法
2. DNS 解析
3. TCP 连接 (三次握手)
4. TCP 连接成功, 发送请求
5. 服务器收到请求, 处理请求并返回响应
6. 浏览器收到 http 响应
7. TCP 连接关闭 (四次挥手)
8. 浏览器解析响应
9. 渲染页面

1. **URL 解析**：浏览器判断输入是搜索词还是 URL
2. **DNS 解析**：

## TCP 连接 (三次握手, 四次挥手)

### TCP连接: 三次握手

1. 第一次握手: 客户端发送 SYN 报文
2. 第二次握手: 服务器发送 SYN ACK 报文
3. 第三次握手: 客户端发送 ACK 报文

### TCP连接关闭: 四次挥手

1. 第一次挥手: 客户端发送 FIN 报文
2. 第二次挥手: 服务器发送 ACK 报文
3. 第三次挥手: 服务器发送 FIN 报文
4. 第四次挥手: 客户端发送 ACK 报文

## 对 DNS 协议的理解

DNS (domain name system): 将域名映射到 IP 上

### 域名解析过程

1. 用户输入域名
2. 检查自身 DNS 缓存
3. 操作系统 DNS 缓存
4. 本地域名服务器
5. 根据本地 DNS 服务器去找根 DNS 服务器, 顶级域名服务器 (TLD), 权威 DNS 服务器
6. 返回结果, 浏览器缓存并向 IP 发送请求

### DNS 记录类型

- A 记录: 将域名映射到 IPv4 域名
- AAAA记录: 将域名映射到 IPv6 地址
- CNAME 记录: 将域名映射到另一个域名
- MX 记录: 指定邮件服务器
- TXT 记录: 文本信息存储, 域名验证 SPF 记录

### DNS 常见问题

- DNS 解析慢: DNS 预解析; 使用 CDN; 减少外部资源请求; 自己的域名 + OSS + CDN
- DNS 劫持: HTTPS 协议; DESSEC (DNS 安全拓展)

### 优化

- DNS 缓存
- nslookup
- dig

## 前端路由模式

### hash 模式

原理:

- URL 结构: Hash 模式的 URL 形如: http://example.com/#/path, # 后面的部分称为哈希值
- 事件监听: 浏览器提供了 hashChange 事件, URL 中的哈希值变化时, 触发该事件 (前端通过监听这个事件实现路由功能)
- 路由匹配: 哈希值变化时, 前端路由会解析新的哈希值, 并根据预定义的路由表来匹配相应的组件或页面
- Hash 模式利用 URL 中的哈希符号 (#) 模拟一个完整的 URL, 实现页面的导航而不重新加载页面
- 哈希值不被包含在 http 请求中 (即哈希值变化, 页面不会重加载, 但浏览器的 URL 会发生变化)

优点:

- 兼容性高, 支持低版本浏览器和 IE 浏览器
- 无需服务端配置 (因为哈希值不会被发送到服务器)

缺点:

- URL 不美观 (URL 中包含 #, 影响 URl 的美观性)
- SEO 不友好 (哈希值不被包含在 http 请求中, 不会被发送到服务器, 搜索引擎爬虫可能无法正确索引这些页面)

### history 模式

原理:

- popState(): 历史记录改变时, 会触发 popState 事件 (监听这个事件实现路由功能)
- pushState(stateObject, title, url): 将新的状态推入历史记录栈, 并可选的改变当前 URL
- replaceState(stateObject, title, url): 替换当前的历史记录栈的状态, 而不是新增一个记录

优点:

- URL 美观: URL 中没有 #
- 更好的用户体验: 用户可以使用浏览器的前进和后退按钮, 不会导致页面重新加载
- SEO 友好: URL 是标准的路径格式, 容易被搜索引擎索引

缺点:

- 需要服务端支持: URl 是真实路径, 服务器需要进行相应配置
- 兼容性问题: 现代浏览器都支持 History API, 部分浏览器可能不支持

## SEO 优化

- 页面结构优化: 语义化标签
- meta, title 优化
- 内容优化: 保证页面中关键词的覆盖率
- 技术项 SEO 优化: robots.txt, 结构化数据, 移动端兼容处理

## clientWidth & clientHeight & clientLeft & clientTop

### clientWidth

- 只读; 元素的内部宽度, 单位为像素
- 包含 content, padding; 不包含 border, margin 及垂直滚动条
- 内联元素的 clientWidth 为 0

### clientHeight

- 只读; 元素的内部高度, 单位为像素
- 包含 content, padding; 不包含 border, margin 及垂直滚动条
- 内联元素的 clientHeight 为 0

### clientLeft

- 只读; 元素左边框的宽度, 单位为像素
- 不包含 padding & margin
- 如果元素的文本方向是从右向左 (RTL, right-to-left), 并且由于内容溢出导致左边出现了一个垂直滚动条, 则该属性包括滚动条的宽度

### clientTop

- 只读; 元素顶边框的宽度, 单位为像素
- 不包含 padding & margin

## offsetWidth & offsetHeight & offsetLeft & offsetTop

### offsetWidth

- 只读; 元素的布局宽度, 单位像素
- 包含 CSS 设置的 width, padding, border 及竖直方向滚动条的宽度
- 各浏览器的 offsetWidth 可能有所不同

### offsetHeight

- 只读; 元素的布局高度, 单位像素
- 包含 CSS 设置的 height, padding, border 及竖直方向滚动条的宽度
- 如果元素被隐藏则返回 0

### offsetLeft

- 只读; 元素左上角相对于父元素左边界的偏移
- 如果元素被隐藏则返回 0

### offsetTop

- 只读; 元素相对于父元素顶部内边距的距离
- 如果元素被隐藏则返回 0

## scrollWidth & scrollHeight & scrollLeft & scrollTop

### scrollWidth

- 只读; 元素内容宽度的度量, 包括由于溢出导致的视图不可见内容
- 没有水平滚动条的情况下, scrollWidth 值与元素视图填充所有内容所需要的最小值 clientWidth 相同

### scrollHeight

- 只读; 元素内容高度的度量, 包括由于溢出导致的视图不可见内容
- 没有垂直滚动条的情况下, scrollHeight 值与元素视图填充所有内容所需要的最小值 clientHeight 相同

### scrollLeft

- 可读可设置; 一个元素的内容水平滚动的像素数 (滚动条到元素左边的距离)

### scrollTop

- 可读可设置; 一个元素的内容垂直滚动的像素数 (滚动条到元素上边的距离)
